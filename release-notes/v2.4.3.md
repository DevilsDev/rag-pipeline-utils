# v2.4.3 — Critical security patch + responsible-disclosure policy

> **Security release.** Patches two CVEs in the runtime dependency tree. Recommended for all consumers of 2.4.x.

## 🔒 Security fixes

### axios upgraded `1.8.4` → `1.15.0`

Closes two **critical** advisories:

- [GHSA-fvcv-3m26-pcqx](https://github.com/advisories/GHSA-fvcv-3m26-pcqx) — Unrestricted cloud metadata exfiltration via header injection chain
- [GHSA-3p68-rc4w-qgx5](https://github.com/advisories/GHSA-3p68-rc4w-qgx5) — SSRF via `NO_PROXY` hostname normalization bypass

### follow-redirects (transitive) patched

[GHSA-r4q5-vmmm-2653](https://github.com/advisories/GHSA-r4q5-vmmm-2653) — moderate auth-header leak on cross-domain redirects.

### Result

`npm audit --omit=dev` now reports **0 vulnerabilities** for runtime dependencies.

## 🛠 CI hardening

- Supply-chain workflow runs `npm audit` with `--omit=dev` (was the deprecated `--production` flag) and scopes the license allowlist to runtime deps only — dev-only licenses no longer break the build, since they don't ship to consumers
- `Unlicense` and `CC-PDDC` added to the license allowlist as legitimate public-domain dedications

## 📜 New: SECURITY.md

Responsible-disclosure policy ships in the tarball, covering:

- **Triage SLAs**: 72 h acknowledgement · 7 d severity rating · 14 d patch for confirmed critical/high
- **Supported versions** matrix
- **Safe-harbor** clause for good-faith research
- **Scope** (what's in vs. out)
- Hardening practices already in place (pinned Actions by SHA, Dependabot, runtime audit gate, license enforcement, SBOM)

GitHub auto-detects `SECURITY.md` and adds private vulnerability reporting via the Security tab.
