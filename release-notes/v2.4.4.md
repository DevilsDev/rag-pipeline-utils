# v2.4.4 — Trust-signal release: benchmarks, stability policy, comparison, demo

> **Docs and examples only.** Zero changes to `src/`. Closes the entire "this-month" repositioning review with five additive deliverables.

## 📊 BENCHMARKS.md (root, ships in tarball)

Methodology-first benchmarks doc. Deliberately does **not** publish a headline "N ops/sec" number. RAG performance depends on provider, hardware, corpus, and queries; one number hides all of that and ages badly.

What you get:

- Philosophy + what we measure (framework overhead) vs. don't (E2E provider-bound)
- Three run presets: `npm run benchmark:quick` / `:full` / default (already in `package.json`)
- JSON report schema + environment disclosure template
- **End-to-end harness template** for honestly benchmarking your own stack
- Interpretation guide (percentiles, throughput, memory, common pitfalls)
- CI regression pattern + community contribution flow

## 📐 SEMVER.md (root, ships in tarball)

Public API stability contract. Defines:

- Public surface = entry-point exports + CLI + plugin contracts in `contracts/`
- What counts as breaking (and what doesn't)
- **Deprecation process**: minimum one-minor warning window before removal
- Supported runtime: Node.js ≥ 18 today, ≥ 20 planned for next major
- TypeScript and ESM/CJS commitments
- Pairs with the Supported Versions matrix in SECURITY.md

You can now auto-merge patch updates with confidence and plan major upgrades from a published contract.

## 🔐 Security Capabilities matrix

New docs page: [Security Capabilities](https://devilsdev.github.io/rag-pipeline-utils/docs/Security-Capabilities)

Honest scorecard distinguishing:

- 🟢 **Implemented** — battle-tested, has tests, in production
- 🟡 **Recommended** — primitives + docs ship, you wire it
- 🟠 **Example** — reference code, audit before production
- 🔴 **Not in scope** — explicitly out of scope

Covers pre-retrieval / retrieval-time / post-generation guardrails plus supply-chain, auth, logging, input validation, and data governance. Use it for vendor-evaluation checklists and threat-model reviews.

## 🚀 Runnable demo: `examples/fastify-rag-demo`

Minimal end-to-end RAG service in **~150 lines of `server.js`**:

- Fastify 5 + OpenAI + in-memory retriever + 3-layer guardrails
- `POST /query` returns `{ answer, citations, groundednessScore, evaluation, retrievedCount }`
- `POST /ingest`, `GET /health`
- CORS + per-IP rate limiting
- **Dockerfile + Fly.io `fly.toml` + Railway `railway.json`** — deploy in one command
- `scripts/smoke.js` for a 30-second end-to-end test

```bash
cd examples/fastify-rag-demo
cp .env.example .env  # set OPENAI_API_KEY
npm install && npm start
node scripts/smoke.js  # ingest + query + assert citations
```

Deploy:

```bash
fly launch && fly secrets set OPENAI_API_KEY=sk-... && fly deploy
# or
railway up
```
