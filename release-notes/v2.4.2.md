# v2.4.2 — Composable RAG repositioning + MIT license

> **Heads-up**: this release contains zero code changes — it's a positioning, license, and docs reset based on community review feedback. Safe to upgrade from any 2.4.x.

## What changed

### 🔓 License: GPL-3.0 → MIT

Removes adoption friction for commercial and closed-source consumers. No more license-compatibility conversations before adoption.

### ✨ Repositioned around three core primitives

The project's framing changed from "104 exports / utility toolkit" to:

> **Composable RAG for Node.js — with built-in evaluation, citations, guardrails, and observability.**

Start with **`pipeline`**, **`plugin`**, **`connector`**. Layer in evaluation, agentic reasoning, and GraphRAG only when you need them.

### 📚 README rewritten

- New **"Who This Is For"** section targeting production Node.js RAG teams
- Three concrete **use-case paths**:
  - Document Q&A with citations
  - Internal knowledge assistant with evaluation
  - Enterprise service with guardrails
- Capabilities table replacing the export-count framing
- ASCII architecture diagram (renders on both GitHub _and_ npm.js, unlike Mermaid)

### 🔧 ESM consistency

Docs-site homepage and Quick Start examples now use ESM `import` everywhere, matching the package's dual-build (`dist/index.mjs` + `dist/index.cjs`).

### 🧹 Removed unverified claims

Docs homepage stat cards no longer claim "<200ms retrieval latency" — replaced with verifiable "MIT License" + "3 Core Primitives". RAG latency is provider-dependent; we now ship a benchmark methodology instead (see v2.4.3+).

### 📦 Tarball

- 542 KB · 221 files
- All-MIT runtime dependency tree

## Migration

No code changes required. If your existing setup works on 2.4.0 or 2.4.1, it works on 2.4.2.
